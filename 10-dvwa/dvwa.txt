I began by installing dvwa on my kali system, navigating to the web directory where I intended to host the files. Using git, I cloned the official dvwa repository from digininja on github and confirmed that the contents were correctly placed in the /var/www/html folder by listing the directory. After adjusting the permissions of the dvwa folder to ensure apache could serve the files, I entered the dvwa directory and located the configuration template config.inc.php.dist. I copied this file, renamed it to config.inc.php, and edited it with moousepad to update the database credentials to admin:password.
Next, I started the mysql service and verified that it was running. Within mysql, I created a new database for dvwa and added a user with the same admin:password credentials to match the configuration file. Once the database was prepared, I turned my attention to apache. I started the apache service, checked its status and modified the php configuration to enable both allow_url_fopen and allow_url_include. After saving the changes, I restarted apache to apply the new settings.
With both mysql and apache running, I accessed dvwa through the browser. On the login page, I entered the admin:password credentials and successfully logged in. From the dvwa homepage, I selected the option to create or reset the database which initialized the dvwa tables. After being redirected back to the login page, I logged in again and was taken to the fully functional dvwa dashboard. Finally, I set the dvwa security level to low to begin practicing basic web exploitation techniques in a controlled environment.
